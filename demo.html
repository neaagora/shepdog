Architecture

Shepdog currently runs as an HTTP MITM proxy — a thin forwarding layer that sits between the agent and its upstream services. Every HTTP request and response passes through it. The proxy generates service records as a structural byproduct: the agent cannot opt out of observation by changing its behavior, because the proxy is not an instrument the agent controls. This is the meaning of observer_independence: "involuntary".

The production shipping format is an MCP server. MCP (Model Context Protocol) is the standard interface through which agents invoke external tools. An MCP-wrapped Shepdog appears to the agent as a normal set of tool definitions — but all invocations pass through a recording layer before reaching the actual service. The agent interacts with the tools normally; the observation is transparent and involuntary. observer_independence: "involuntary" is preserved in both deployment modes: the proxy intercepts at the HTTP layer, the MCP server intercepts at the tool-call layer.

The three documented failure modes — Helpful Lie, Hallucinated Tool Use, and Empty Success Trap — are detected mechanically through pattern matching on the call graph, without LLM judgment. The detector does not need to understand what the agent was "trying" to do. It only needs to compare the observable call record against the agent's completion claims.
